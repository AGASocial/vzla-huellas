# i18n ES/EN — Design Spec

**Fecha:** 2026-06-30

## Objetivo

Agregar soporte bilingüe español/inglés con detección automática del idioma del navegador. URLs incluyen el locale: `/es/...` y `/en/...`. Selector 🇻🇪 ES | 🇺🇸 EN visible en todas las páginas.

---

## Arquitectura

**Librería:** `next-intl` (estándar para App Router)

**Estructura de archivos nueva:**
```
messages/
  es.json              ← todas las strings en español
  en.json              ← todas las strings en inglés
src/
  i18n/
    routing.ts         ← define locales ["es", "en"], defaultLocale "es"
    request.ts         ← config de next-intl para server components
  app/
    [locale]/          ← NUEVO: carpeta dinámica que captura el locale
      layout.tsx       ← layout principal (mueve el actual aquí)
      page.tsx
      base-datos/
      escanear/
        guia/
      familiares/
        nuevo/
        [id]/candidatos/
      huellas-desconocidas/
        [id]/candidatos/
  components/
    LocaleSwitcher.tsx ← NUEVO
middleware.ts          ← NUEVO: detección automática y redirect
next.config.ts         ← actualizar con createNextIntlPlugin
```

**Flujo de routing:**
1. Usuario llega a `/` → middleware lee `Accept-Language` header
2. Si es `es-*` → redirige a `/es`; cualquier otro → `/en`
3. El selector manual en el header navega al mismo path en el otro locale

---

## Componente LocaleSwitcher

Ubicado en el header del layout (arriba del stripe de bandera venezolana o dentro del `<body>`). Muestra:

```
🇻🇪 ES  |  🇺🇸 EN
```

El locale activo va en negrita/resaltado. Al hacer click en el otro, navega a la misma ruta pero con el locale opuesto (ej. `/es/base-datos` → `/en/base-datos`).

Usa `usePathname()` de `next-intl/client` y `Link` de `next-intl` para construir la URL alternativa.

## Navegación interna

`next-intl` provee versiones de `Link` y `useRouter` que incluyen el locale automáticamente. Se usan en reemplazo de los imports de `next/link` y `next/navigation` en todos los archivos bajo `[locale]/`.

```typescript
// En vez de:
import Link from "next/link"
import { useRouter } from "next/navigation"

// Usar:
import { Link } from "@/i18n/navigation"
import { useRouter } from "@/i18n/navigation"
```

Se crea `src/i18n/navigation.ts` que re-exporta estos con el config de locales.

## metadata

El `layout.tsx` exporta `metadata` con título y descripción localizados (es/en).
